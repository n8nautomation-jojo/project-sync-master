

# تقرير تقني شامل واحترافي — نظام إدارة التحويلات والإيرادات

---

## أولاً: الوضع الحالي للنظام

### نسبة الاكتمال: ~75%

النظام يعمل كمنصة استلام إشعارات تحويلات عبر واتساب مع معالجة بالذكاء الاصطناعي. الجزء المبني متين تقنياً، لكنه يغطي فقط **جانب الإيرادات** ولا يشمل المصروفات أو المحاسبة الشاملة.

## ثالثاً: الثغرات الأمنية المكتشفة 🔴

### 1. تسريب مفاتيح واتساب (خطورة عالية)
جدول `whatsapp_connections` يحتوي على `access_token` و `green_api_token` — أي عضو في المنظمة (حتى viewer) يمكنه قراءتها.
**الحل:** إنشاء جدول منفصل `whatsapp_credentials` مقيد بـ owner/admin فقط.

### 2. تصعيد الصلاحيات (خطورة عالية)
سياسة INSERT في `user_roles` تسمح للـ admin بمنح نفسه صلاحية `owner`.
**الحل:** إضافة شرط `WITH CHECK` يمنع إنشاء دور أعلى من دور المستخدم الحالي.

### 3. Realtime بدون تقييد (خطورة متوسطة)
أي مستخدم مسجل يمكنه الاشتراك في قنوات إشعارات مستخدمين آخرين.
**الحل:** إضافة سياسة RLS على اشتراكات Realtime.

**الإجابة: ليس بعد.** يحتاج إصلاح الثغرات الأمنية الثلاث أولاً (30 دقيقة عمل)، وإضافة صفحة سياسة الخصوصية، وتفعيل تأكيد البريد. بعدها يمكن نشره كـ "إصدار أولي" لجانب الإيرادات فقط.

## سادساً: خارطة طريق نظام المصروفات — التصميم المعماري

### الرؤية
تحويل النظام من "متتبع إيرادات واتساب" إلى **نظام محاسبي متكامل** يشمل: الإيرادات (التحويلات الحالية) + المصروفات + الرواتب + التقارير المالية الشاملة.

### البنية المقترحة لقاعدة البيانات

```text
┌─────────────────────────────────────────────────────┐
│                  organizations                       │
│  (الكيان الأساسي — كل شيء يتبع لمنظمة)              │
└──────────┬──────────────────────────┬────────────────┘
           │                          │
    ┌──────▼──────┐          ┌────────▼────────┐
    │  transfers  │          │   expenses      │
    │  (إيرادات)  │          │   (مصروفات)     │
    └─────────────┘          └────────┬────────┘
                                      │
                             ┌────────▼────────┐
                             │expense_categories│
                             │ (فئات المصروفات) │
                             └─────────────────┘

    ┌─────────────┐          ┌─────────────────┐
    │  employees  │          │  salary_payments │
    │  (موظفين)   │──────────│   (الرواتب)      │
    └─────────────┘          └─────────────────┘
```

### الجداول الجديدة المطلوبة

**1. `expense_categories` — فئات المصروفات (قابلة للتخصيص)**
- `id`, `organization_id`, `name`, `icon`, `color`, `is_system` (فئات افتراضية مثل: إيجار، كهرباء، مواد خام)
- يمكن للمحاسب إنشاء فئات مخصصة حسب طبيعة عمله

**2. `expenses` — المصروفات**
- `id`, `organization_id`, `branch_id`, `category_id`, `amount`, `description`, `expense_date`, `receipt_image_url`, `is_recurring`, `recurrence_type`, `created_by`, `approved_by`, `status` (pending/approved/rejected)

**3. `employees` — الموظفين**
- `id`, `organization_id`, `branch_id`, `full_name`, `position`, `base_salary`, `hire_date`, `phone`, `is_active`

**4. `salary_payments` — سجل الرواتب**
- `id`, `employee_id`, `organization_id`, `month`, `year`, `base_amount`, `deductions`, `bonuses`, `net_amount`, `paid_at`, `paid_by`

**5. `financial_summaries` — ملخصات مالية شهرية (view أو materialized)**
- إجمالي الإيرادات، إجمالي المصروفات، صافي الربح، مقارنة بالشهر السابق

### الصفحات الجديدة المطلوبة

| الصفحة | الوصف |
|--------|-------|
| `/expenses` | قائمة المصروفات مع فلترة بالفئة والفرع والتاريخ |
| `/expenses/categories` | إدارة فئات المصروفات (إضافة/تعديل/حذف) |
| `/employees` | قائمة الموظفين وبياناتهم |
| `/salaries` | سجل الرواتب الشهري مع إمكانية الدفع الجماعي |
| `/financial-reports` | تقرير شامل: إيرادات + مصروفات + رواتب = صافي الربح |

### التكامل التقني بين الإيرادات والمصروفات

- صفحة Dashboard تعرض: إيرادات اليوم + مصروفات اليوم + الصافي
- صفحة Reports تُدمج البيانات من `transfers` + `expenses` + `salary_payments`
- الإحصائيات تضيف رسوم بيانية جديدة: مقارنة الإيرادات vs المصروفات شهرياً
- التنبيهات: إشعار عندما تتجاوز المصروفات نسبة معينة من الإيرادات

## سابعاً: خطة التنفيذ المرحلية

### المرحلة 0 — إصلاح الأمان (يجب قبل النشر)
1. فصل مفاتيح واتساب في جدول مقيد
2. منع تصعيد الصلاحيات في user_roles
3. إضافة صفحة سياسة خصوصية
4. تفعيل تأكيد البريد الإلكتروني

### المرحلة 1 — نظام المصروفات الأساسي
1. إنشاء جداول `expense_categories` و `expenses` مع RLS
2. بناء صفحة المصروفات مع إضافة/تعديل/حذف
3. بناء صفحة فئات المصروفات
4. تحديث Dashboard لعرض المصروفات بجانب الإيرادات

### المرحلة 2 — نظام الرواتب
1. إنشاء جداول `employees` و `salary_payments`
2. بناء صفحة الموظفين
3. بناء صفحة الرواتب مع دفع جماعي شهري

### المرحلة 3 — التقارير المالية الشاملة
1. تطوير صفحة تقارير مالية موحدة (إيرادات - مصروفات - رواتب = صافي)
2. إضافة تصدير PDF/Excel للتقارير الشهرية
3. إضافة رسوم بيانية مقارنة شهرية

### المرحلة 4 — ميزات مبتكرة مستقبلية
- تنبيهات ذكية (تجاوز ميزانية، مصروف غير معتاد)
- تكرار المصروفات الثابتة تلقائياً (إيجار شهري)
- لوحة تحكم مالية للمدير التنفيذي
- تطبيق هاتف (PWA محسّن)
- ربط مع بنوك سودانية عبر API
- نظام فواتير إلكترونية
