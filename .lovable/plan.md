## المشكلة

الملف `src/integrations/supabase/client.ts` مُولَّد تلقائياً بواسطة Lovable Cloud وممنوع تعديله. أحد الأدوار السابقة أضاف داخله كتلة "fail-loud" تفعل التالي عند غياب متغيري `VITE_SUPABASE_URL` أو `VITE_SUPABASE_PUBLISHABLE_KEY`:

1. تستبدل كامل `document.body.innerHTML` برسالة عربية.
2. ترمي استثناءً وقت استيراد الوحدة (module top-level)، أي قبل تركيب React، فلا يمسكه `ErrorBoundary`.

نتيجة ذلك على النشر: شاشة بيضاء + الرسالة العربية "متغيرات البيئة التالية غير موجودة في بيئة البناء" — وهذا ما يراه المستخدم.

المتغيرات موجودة في `.env` بالمستودع (`VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `VITE_SUPABASE_PROJECT_ID`). في المعاينة الحالية يعمل التطبيق بلا مشاكل — الكسر يظهر فقط في بيئات البناء التي يعيد فيها Lovable توليد `client.ts` بصيغته القياسية أو التي يتفاعل فيها الحقن مع هذه الحراسة بشكل غير متوقع.

## الحل

إعادة `src/integrations/supabase/client.ts` إلى صيغته القياسية المُولَّدة تلقائياً فقط، دون أي منطق إضافي:

```ts
// This file is automatically generated. Do not edit it directly.
import { createClient } from '@supabase/supabase-js';
import type { Database } from './types';

const SUPABASE_URL = "https://uwtcjimjulmhazybmvyd.supabase.co";
const SUPABASE_PUBLISHABLE_KEY = "eyJhbGciOi...";  // القيمة القائمة من .env

export const supabase = createClient<Database>(SUPABASE_URL, SUPABASE_PUBLISHABLE_KEY, {
  auth: {
    storage: localStorage,
    persistSession: true,
    autoRefreshToken: true,
  }
});
```

استخدام قيم ثابتة (وليس `import.meta.env`) هو النمط الرسمي الذي يستخدمه Lovable Cloud تحديداً لتجنّب اعتماد البناء على متغيرات بيئة قد لا تُحقن، ولأن `PUBLISHABLE_KEY` مفتاح عام (anon) يمكن أن يعيش في الكود.

## ما لن يتم لمسه

- لا تعديل على `.env` (auto-generated).
- لا تعديل على `types.ts` (auto-generated).
- لا تعديل على أي كود منطقي/واجهة/hooks/edge functions.
- لا تغيير في السلوك داخل المعاينة الحالية (تعمل الآن).

## التحقق

بعد التطبيق:
1. تشغيل `playwright` على `/`, `/auth`, `/dashboard` للتأكد من عدم ظهور أخطاء وحدة تحكم أو شاشة بيضاء.
2. طلب من المستخدم إعادة النشر (Publish → Update) والتأكد من أن الرسالة العربية لم تعد تظهر.
